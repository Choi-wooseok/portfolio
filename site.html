  • SNS의 특성상 한 페이지에 많은 기능을 필요로 하기때문에 대부분의 코드가 javascript로 이루어져있습니다.
  • MainFeed는 비회원도 볼 수 있는 페이지로 저희 사이트의 메인페이지입니다.
  • Infinity Scroll을 이용하여 페이징처리 하였으며 ‘Simple is the Best’를 모토로 카드형식의 Masonry layout을 이용하여 깔끔한 UI를 구현하고자 했습니다.
  • 카드를 클릭 시 DetailFeed가 모달창으로 나오게 됩니다.
  • DetailFeed에서 이미지는 여러 개의 경우 이미지 우측 하단에 아이콘으로 표시해주고 Drag 방식의 페이징이 가능하게 했습니다.
  • DetailFeed의 화면 우측 상단은 게시글이, 하단에는 댓글이 표시되도록 구현했고 로그인 시에만 댓글 등록과 좋아요 기능을 사용 가능하도록 제한을 두었습니다.
  • KeyUp Event를 사용하여 사용자의 입력값에 즉각적으로 반응할 수 있게 구현했습니다.
  • Event 함수 안에 조건을 주어 #을 먼저 입력할 경우 해시태그만, 그냥 입력할 경우 회원 닉네임이 먼저 나오고 밑에 해시태그가 뜰 수 있게 구현했습니다.
  • 검색 결과에서 닉네임을 클릭 시 해당 회원의 프로필 페이지로 이동하고 해시태그를 클릭 시 해당 해시태그가 있는 모든 게시물들이 메인화면 대신 뜨게 됩니다.
  • 해시태그 검색결과 화면의 경우 화면을 따로 구현하지 않고 메인페이지를 뿌려주는 Ajax에서 조건을 주어 검색어가 들어올 경우 해당 검색어가 존재하는 카드만 뿌려줄 수 있게 구현하여 통일성을 주고자 했습니다.
  • 검색과 마찬가지로 KeyUp Event함수 내에 조건을 주어 게시글이나 댓글 작성시 #이 들어오면 다음 띄어쓰기까지를 해시태그 블록으로 표시해줍니다.
  • 글 작성 이후에는 해당 글을 뿌려줄 때 #을 감지하여 해시태그를 span태그로 감싸주고 span태그에 ClickEvent를 주어 해시태그 클릭 시 검색된 화면으로 이동할 수 있게 구현했습니다.
  • 댓글은 DetailFeed에서 확인 혹은 작성할 수 있게 구현했습니다.
  • DetailFeed 전체가 javascript로 되어있기 때문에 댓글 또한 마찬가지로 javascript로 구현되어있습니다.
  • 비회원의 경우 댓글은 읽기만 가능합니다.
  • 로그인시 댓글작성 및 좋아요, 신고가 가능하며 댓글 작성시 해시태그도 작성 가능합니다.

  • 커뮤니티의 기능 중 팀프로젝트를 대비한 게시판입니다.
  • 3차로 나뉘어진 프로젝트의 특징을 기준으로 구현했습니다.
  • 상단에는 현재페이지의 프로젝트 차수 및 조 번호 navigation이 위치해있습니다.

  • 글 제목 클릭시 detail modal창이 뜨며 내용을 확인할 수 있고 댓글보기 클릭시 댓글창이 아코디언 형식으로 나타나게 됩니다.
